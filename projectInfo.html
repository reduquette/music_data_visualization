About Our Project

Data Processing

Once we found our data sets from Kaggle on Rolling Stone Top 500 Songs of All Time and Spotify Tracks DB, we had to process the data using python to combine the data sets. We used the python package pandas to read the csv files and then used the re package to remove any special characters. The songs were listed with different capitalization, spacing, and wording between the data sets to reference the same song. So, we took out spaces, made all letters lowercase, and checked for the word "remastered" appended to the end of each song to identify 442 song matches common to the two data sets.

Design Processes

Low Fidelity Prototype: We looked through the different attributes of the data and decided on which we found most interesting. We then sketched out different visualization ideas that we had for these different attributes and went through a selection process of which visualizations would most help the user decide if modern music sucks or not. We had to then rework the data to be workable with what we wanted to portray with these visualizations.

High Fidelity Prototype: Once the data was reworked to be more usable, we each took on a visualization. We made sure to organize the visualizations so that they could be compared by decade. We first represent the overall comparison of the most popular songs; then we show the producers of these popular songs before going deeper into the actual features of the songs (going from large to small scale).

User Review: There were many comments on the scrolling ability of the visualizations from all the users which we fixed by switching the site from a vertical scroll to a horizontal. Simplification was another suggestion we recieved as well as more explanation of the visualizations themselves. With the change to Horizontal scrolling we also stripped the amount of items on each page as well as changed the colors to further simplify while also making the site easier to go through.
